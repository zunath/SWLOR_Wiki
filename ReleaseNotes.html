Release Page Format

  • Use ReleaseNotes/RN_Unreleased for draft notes that are still being collected.
  • Use one child page per finalized release under ReleaseNotes/RN_YYYY-MM-DD.
  • Use a suffix for distinct releases on the same date, such as ReleaseNotes/RN_YYYY-MM-DD-Hotfix.
  • Start each page with the release date and NWSync hash. Use TBD until those values are known, or Not listed when importing historical notes without a hash.
  • Credit contributors near the top when available, without Discord mention formatting.
  • Group notes by topic when a release has many changes or source parts.
