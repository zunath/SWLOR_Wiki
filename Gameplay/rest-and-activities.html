Rest, busy states, and downtime

Rest (camp / rest action)

SWLOR replaces the default Neverwinter Nights rest finish with a custom Rest status effect. When you start resting (after the server accepts the rest attempt):

  • You play a seated loop animation and must stay within a small area; moving more than a tiny amount ends rest.
  • Your companion (henchman) can receive the same rest effect alongside you.
  • Over time, the effect ticks heal HP and restore stamina and force. Amounts scale with Vitality (with minimums so low scores still recover). Active food buffs that boost rest regeneration add to those ticks.
  • Taking damage or starting an attack removes rest.
  • Logging in clears rest.

Rest ticks run on the server’s status-effect heartbeat (not on the vanilla NWN “rest completed” timer).

Where you may rest

  • Open world / non-dungeon areas: you can rest anywhere that passes the safety checks below.
  • Dungeon areas (areas flagged as dungeons): you may only rest inside designated rest triggers. Entering one messages that it looks safe; leaving clears that permission.

You cannot rest while you (or a party member within range) are in combat, while you are personally in combat, or while an enemy is nearby (roughly within 20 meters). The server returns a clear error string in those cases.

Busy flag

The server sets an internal busy state for focused actions. While busy, some systems refuse new actions (for example many item activations) with “You are busy.” Examples that set busy include:

  • Resting
  • Ability activation (during the window the ability uses)
  • Using items with the busy guard
  • Fishing
  • Certain quest item turn-in flows

Busy is cleared when those actions finish (for example rest ends, container turn-in closes) and is wiped on character login or death so stuck states do not persist.

Quest journal

Abandoning a quest from the Quests UI also checks busy; finish resting or other busy interactions first.
